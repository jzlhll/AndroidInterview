阶段执行顺序：组合 → 布局 → 绘制，不可逆，前一阶段未完成不会进入下一阶段；
副作用执行时机：
LaunchedEffect/SideEffect：组合阶段完成后、布局 / 绘制前执行；
DisposableEffect：组合阶段完成后立即执行，清理逻辑在重组前 / 组合离开时执行；
性能优化核心：
用 remember 缓存计算结果，减少重组开销；
用 derivedStateOf 过滤高频状态变化，避免无效重组；
避免在布局 / 绘制阶段执行耗时操作（如复杂计算）。
总结
Compose 渲染分为组合、布局、绘制三个单向执行的阶段，状态变化仅触发受影响的阶段重执行；
组合阶段构建 UI 结构，布局阶段计算尺寸 / 位置，绘制阶段渲染视觉效果；
副作用需绑定阶段生命周期执行，性能优化的核心是最小化各阶段的重执行范围。
